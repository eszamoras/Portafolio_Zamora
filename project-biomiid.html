Live-Cell Microscopy
Data Acquisition System

C Arduino Mega I²C / TWI AD5669 16-bit DAC NI PCIe-6353 NI-DAQmx NIS-Elements 6.20 TTL Triggering
◎ Overview

A precision real-time data acquisition system developed for Vanderbilt University's BioMIID Lab, supporting live-cell microscopy research. The system acquires environmental data from four sensors — temperature, pH, humidity, and volumetric water content — via I²C on an Arduino Mega, encodes each reading into a calibrated analog voltage through a 16-bit AD5669 DAC, and routes those signals into a National Instruments PCIe-6353 DAQ card for real-time display in a NIS-Elements 6.20 GUI with live time-series plots. The full signal chain was validated through structured integration testing, oscilloscope verification, and systematic fault isolation across firmware, hardware, and software layers.

⊙ System Objective

Provide lab researchers with a real-time, calibrated, multi-sensor environmental monitoring pipeline that integrates seamlessly into the existing NIS-Elements microscopy workflow — zero manual data recording, live visualization, and hardware-verified signal accuracy.

⊘ Design Constraints
All four sensors must share a single I²C bus on the Arduino Mega; address conflicts must be resolved at the firmware layer
AD5669 DAC output voltage must correspond exactly to the physical sensor range; encoding/decoding equations must be symmetric and verified against multimeter measurements
NI-DAQmx analog input channel assignments must match DAC output port wiring exactly — channel mismatch causes silent measurement errors
TTL pulse timing on digital I/O port 2 must align with NIS-Elements trigger expectations to synchronize acquisition with microscopy events
Safety timeout mechanism must not interfere with valid data transmission — must be removed or redesigned if it causes false-positive data corruption flags
⊞ System Architecture
Full signal chain: Sensors → Arduino Mega → AD5669 DAC → NI PCIe-6353 → NIS-Elements GUI
① NIDAQ ② Power Rails ③ I²C / Analog ④ DAC Output ⑤ Relay
⊛ Firmware Architecture

The Arduino Mega firmware operates a continuous I²C polling loop across four sensor addresses. For each sensor, the raw reading is acquired, scaled to a voltage within the DAC's 0–5 V output range using a pre-characterized linear equation (e.g., temperature mapped −20 °C to 120 °C → 0 V to 5 V), and written to the corresponding AD5669 channel via SPI. A diagnostic mode was added that prints raw sensor values, expected DAC voltages, and measured DAC voltages to the serial monitor simultaneously, enabling direct comparison during validation. The TEROS 10 volumetric water content sensor is powered via a relay triggered by Arduino GPIO, activated only during the measurement window to prevent continuous power draw. A TTL pulse is sent on digital I/O port 2 to synchronize the NIS-Elements acquisition trigger with each sensor polling cycle.

◈ Determinism & Hard Real-Time Guarantees

The polling loop executes at a fixed cadence with no OS preemption. Each I²C transaction has a bounded worst-case duration determined by the sensor's response time and the bus clock rate. DAC SPI writes complete in a deterministic number of clock cycles. The TTL trigger pulse width is enforced by a NOP-calibrated delay to meet NIS-Elements timing requirements. The original safety-timeout mechanism that forced DAC output to 0 V after 2 seconds was identified as a source of false data corruption flags and removed — its behavior was non-deterministic relative to valid sensor polling cycles. Channel-to-port assignments are hardcoded and verified against the NI-DAQmx configuration, eliminating runtime channel resolution.

🔍 Fault Isolation Log — Real Engineering, Real Bugs
BUG #1 · FIRMWARE
Safety Timeout Corrupting Valid Data
A firmware safety mechanism forced DAC output to 0 V after 2 seconds when data was flagged as potentially corrupted. The flag was incorrectly triggering on valid sensor data, resetting outputs to zero mid-acquisition. Identified by reviewing firmware line-by-line against expected NI-DAQ readings. Removed the timeout — NI-DAQ immediately began displaying correct values.
BUG #2 · HARDWARE
DAC Internal Reference Causing 1.5× Voltage Gain
Measured DAC output voltages were consistently ~1.5× higher than expected. Isolated by commanding the DAC to output known fixed voltages and measuring with a multimeter directly at the output pins, ruling out cable/connector faults. Traced to an internal reference enable bit in the firmware that overrode the intended 5 V external reference. Removed the configuration line; DAC voltages matched expected values within <2% error.
BUG #3 · HARDWARE
RTD Probe Shield Blocking Thermal Response
Temperature sensor read a constant value regardless of environment. Firmware and electrical connections were verified correct. Physical inspection of the RTD probe revealed an improperly installed protective shield blocking thermal contact. Removing the shield immediately restored correct temperature response, verified against a reference thermometer.
BUG #4 · NI-DAQ
Channel Mismatch: AI14 → AI15
NI-DAQ displayed temperature values such as −19 °C or −20 °C despite correct firmware output. Suspected channel assignment error. Switching the NI-DAQmx input from AI14 to AI15 immediately resolved the discrepancy — values matched sensor readings. Confirmed by oscilloscope trace on the correct channel.
BUG #5 · SCALING
Unit Mismatch: °F Firmware vs °C NI-DAQ
NI-DAQ temperature display was offset from expected values due to a unit mismatch — firmware encoded in °F while NI-DAQ decoded in °C. Resolved by recalibrating the sensor output to °C and updating both the firmware scaling equation and the NI-DAQ decoding equation to match, ensuring encoding/decoding symmetry across the full signal chain.
VALIDATION · SYSTEM TEST
Full System Integration Test Passed
Final validation: plant mounted on Smartpot with all four sensors active. Perturbed temperature (hot/cold water) and VWC (soil moisture changes) while monitoring serial monitor and NIS-Elements GUI simultaneously. All sensors responded correctly to perturbations. VWC readings consistent with expected ~40 m³/m³ for potted soil. System confirmed operational for live-cell microscopy research use.

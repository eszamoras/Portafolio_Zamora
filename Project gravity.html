14-Body Gravitational Simulation

C++OOPDesign PatternsCI/CDGitclang-format
◎ Overview

A 14-body solar system simulation in C++ applying Singleton, Visitor, Factory, and Prototype design patterns. Earth's full year-long orbit is reproduced with less than 1,000 km positional error over 31.5 million time steps, validated by a 5-test automated suite with CI enforcement.

⊙ System Objective

Implement a physically accurate N-body gravitational simulation of the solar system in modern C++ using canonical software design patterns. Validate correctness through a rigorous automated test suite and enforce code quality via CI/CD pipelines.

⊘ Design Constraints
Singleton pattern must ensure exactly one PhysicsEngine instance exists across the application lifetime
Factory pattern must instantiate all 14 bodies (Sun + 8 planets + 5 dwarf/notable bodies) from a configuration source
Visitor pattern must allow extensible per-body operations without modifying body classes
Prototype pattern must support deep-copy body instantiation for scenario branching
Earth's orbital position must stay within 1,000 km of reference ephemeris at every 100-second interval
All commits must pass clang-format linting; CI must reject non-conforming code

⊛ Software Architecture

The C++ architecture centers on a PhysicsEngine Singleton owning the simulation loop. A BodyFactory reads initial conditions (position, velocity, mass) and constructs CelestialBody objects. Each time step the engine iterates all body pairs, computes the Newtonian force vector F = G·m₁·m₂/r³ · (r₂−r₁), and accumulates it onto both bodies (Newton's third law). Forces integrate into velocities (v += F/m · Δt) and velocities into positions (x += v · Δt) via symplectic Euler. The Visitor pattern applies post-step operations (energy logging, position output) per-body without coupling to the Body class. Prototype's clone() snapshots state for test comparison.

◈ Correctness & Validation

The simulation is deterministic by design — given fixed initial conditions and a fixed timestep, the same trajectory is always produced. IEEE 754 double-precision floating point ensures reproducible results across platforms. The 1-second timestep over 31.5 million steps (≈1 year) was selected to balance integration error against execution time, achieving sub-1,000 km positional accuracy for Earth's orbit.
